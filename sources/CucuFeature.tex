##########################################
Feature: Remove project activity # Emil 1
  Description: The act of removing a project activity
  Actor: Project manager

  Scenario: Remove an existing project activity when you are the project manager for the project containing said activity
    Given there is a project with name "project1"
    And the project manager for "project1" is "Pete"
    And "project1" contains an activity "activity1"
    When the project manager tries to remove "activity1"
    And the project manager states that the activity should be removed from "project1"
    And the project manager states that "Pete" is the one doing the action
    Then the activity is removed 
    And "project1" no longer contains an activity called "activity1"

  Scenario: Remove an existing project activity when you are not the project manager for the project containing said activity
    Given there is a project with name "project2"
    And the project manager for "project2" is "Gabe"
    And "project2" contains an activity "activity2"
    When the project manager tries to remove "activity2"
    And the project manager states that the activity should be removed from "project2"
    And the project manager states that "Mike" is the one doing the action
    Then the actor should be informed the the action has to be done by the project manager
    And the activity is not removed
    And "project2" still contains an activity called "activity2"

  Scenario: Remove a project activity from a project that does not contain said activity 
    Given there is a project with name "project3"
    And the project manager for "project3" is "John"
    And "project3" does not contain an activity "activity3"
    When the project manager tries to remove "activity3"
    And the project manager states that the activity should be removed from "project3"
    And the project manager states that "John" is the one doing the action
    Then the actor should be informed that "activity3" does not exist
    And no project is removed

  Scenario: Remove a project activity from a project that does not exist
    Given no project called "project4" exists
    When the actor tries to remove "activity4" from "project4"
    Then the actor should be informed that the given project does not exist
    And no project activity is removed

####  Assign employee to an activity  ####

Feature: Assign an employee to a project-activity # Emil 2
  Description: The act of assigning an employee to a project-activity
  Actor: project manager

  Scenario: Assign an employee to an activity in a project when the employee has not exceeded 19 activities for the time of the project and you are the project manager for the project
    Given there is a project with name "project1"
    * the project manager for "project1" is "Adam"
    * the project contains an activity "activity1"
    * there exists an employee "Fred"
    * "Fred" has 19 assigned activities for the weeks assigned to "activity1"
    * "Fred" is not assigned to "activity1"
    When the project manager tries to assign "Fred" to "activity1" in "project1"
    And states that the actor is "Adam"
    Then "Fred" is assigned to "activity1"
    And "Fred" has 20 assigned activities for the weeks assigned to "activity1"

  Scenario: Assign an employee to an activity in a project when the employee has exceeded 19 activities for the time of the project
    Given there is a project with name "project2"
    * the project manager for "project2" is "Adam"
    * the project contains an activity "activity2"
    * there exists an employee "Fred"
    * "Fred" has 20 assigned activities for the weeks assigned to "activity2"
    * "Fred" is not assigned to "activity2"
    When the project manager tries to assign "Fred" to "activity2" in "project2"
    And states that the actor is "Adam"
    Then the project manager is informed that the employee is already assigned 20 activities for the given time
    And "Fred" is not assigned to "activity2"
    And "Fred" has 20 assigned activities for the weeks assigned to "activity2"

  Scenario: Assign an employee to an activity where you are not the project manager for the project
    Given there is a project with name "project3"
    * the project manager for "project3" is "Adam"
    * the project contains an activity "activity3"
    * there exists an employee "Fred"
    * "Fred" has 19 assigned activities for the weeks assigned to "activity3"
    * "Fred" is not assigned to "activity3"
    When the project manager tries to assign "Fred" to "activity3" in "project3"
    And states that the actor is "Dave"
    Then the project manager is informed that the user "Dave" is not project manager for "project3" and can not assign employees to the given activity
    And "Fred" is not assigned to "activity3"
    And "Fred" has 19 assigned activities for the weeks assigned to "activity3"

  Scenario: Assign an employee to an activity that does not exist
    Given there is a project with name "project4"
    * the project manager for "project4" is "Adam"
    * the project does not contain an activity "activity4"
    * there exists an employee "Fred"
    When the project manager tries to assign "Fred" to "activity4" in "project4"
    And states that the actor is "Adam"
    Then the project manager is informed that the given activity does not exist
    Then "Fred" is not assigned to "activity4"
    And "Fred" has 19 assigned activities for the weeks assigned to "activity4"

  Scenario: Assign an employee to an activity in a project that does not exist
    Given no project named "project5" exists
    * there exists an employee "Fred"
    When the project manager tries to assign "Fred" to an activity in "project5"
    Then the project manager is informed that the given project does not exist
    And "Fred" is not assigned to an activity
  
  Scenario: Assign an employee to an activity in a project that the given employee is already assigned to
    Given there is a project with name "project6"
    * the project manager for "project6" is "Adam"
    * the project contains an activity "activity6"
    * there exists an employee "Fred"
    * "Fred" has 19 assigned activities for the weeks assigned to "activity6"
    * "Fred" is assigned to "activity6"
    When the project manager tries to assign "Fred" to "activity6" in "project6"
    And states that the actor is "Adam"
    Then the project manager is informed that the employee is already assigned to the activities
    And "Fred" has 19 assigned activities for the weeks assigned to "activity6"
    And "Fred" is assigned to "activity6" once
    But "Fred" is not assigned to "activity6" twice

  Scenario: Assign an employee that does not exist to an activity
    Given there is a project with name "project7"
    * the project manager for "project7" is "Adam"
    * the project contains an activity "activity7"
    * there exists no employee "Fred"
    When the project manager tries to assign "Fred" to "activity7" in "project7"
    And states that the actor is "Adam"
    Then the project manager is informed that the given employee does not exist
    And no employee "Fred" exists
    And no employee is assigned to "activity7"
